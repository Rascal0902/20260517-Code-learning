# Cube Solving

## 정의
이 문제는 현재 큐브 상태에서 시작하여, 정답 상태까지 가는 가장 짧은 동작 순서를 찾는 문제이다.  
즉, 큐브를 풀 수 있는 상태라고 가정했을 때, 주어진 `18개의 action`만 사용해서 최소 몇 번의 움직임으로 풀 수 있는지를 구하고, 그 실제 동작 순서를 출력하는 것이 목표이다.

여기서 `18개의 action`은 보통 각 면에 대해 가능한 회전을 모두 포함한 동작 집합을 뜻한다. 예를 들어 각 면에 대해 시계 방향 `90도`, 반시계 방향 `90도`, `180도` 회전이 가능하다고 보면 총 `6 x 3 = 18`개의 action이 된다.

## 문제 상황
큐브의 한 상태가 입력으로 주어진다. 이 상태는 이미 섞여 있지만, 풀 수 없는 상태가 아니라 반드시 정답 상태로 갈 수 있다고 가정한다.

이때 해야 할 일은 다음과 같다.

- 현재 상태에서 시작한다.
- 한 번에 할 수 있는 행동은 `18개 action`뿐이다.
- 가능한 여러 경로 중에서 가장 적은 횟수로 풀리는 경로를 찾아야 한다.
- 마지막에는 몇 번 만에 풀리는지와 어떤 순서로 움직이면 되는지를 출력해야 한다.

즉, 단순히 "풀 수 있다"를 확인하는 것이 아니라, "가장 빨리 푸는 방법"을 찾는 문제이다.

## 알고리즘
이 문제는 그래프 최단경로 문제처럼 볼 수 있다.

- 하나의 큐브 상태를 하나의 정점으로 본다.
- 어떤 상태에서 `18개 action` 중 하나를 적용해 다른 상태로 갈 수 있으면, 두 상태 사이에 간선이 있다고 본다.
- 모든 action의 비용을 `1`로 보면, 최소 횟수 해법은 최단경로를 찾는 문제와 같아진다.

이 경우 가장 기본적인 방법은 `BFS(너비 우선 탐색)`이다.

### 진행 방법

1. 시작 큐브 상태를 큐에 넣는다.
2. 시작 상태가 이미 정답 상태인지 먼저 확인한다.
3. 큐에서 상태 하나를 꺼낸다.
4. 그 상태에서 가능한 `18개 action`을 모두 적용하여 다음 상태들을 만든다.
5. 아직 방문하지 않은 상태라면 방문 처리하고 큐에 넣는다.
6. 이때 "어떤 상태에서 왔는지"와 "어떤 action을 사용했는지"를 함께 기록한다.
7. 정답 상태가 나오면 탐색을 멈춘다.
8. 기록해 둔 이전 상태와 action을 거꾸로 따라가면 최소 동작 순서를 복원할 수 있다.

### 왜 BFS를 쓰는가
모든 action의 비용이 같기 때문이다.  
한 번 움직일 때마다 비용이 모두 `1`이라면, BFS는 가장 먼저 도달한 정답 상태가 곧 최소 이동 횟수 해법임을 보장한다.

### 출력
알고리즘이 끝나면 보통 다음 내용을 출력한다.

- 최소 action 수
- action의 실행 순서

예를 들면 다음과 같은 형태가 된다.

- 최소 횟수: `5`
- 동작 순서: `U, R', F2, L, D`

## 구현할 때 필요한 정보
이 문제를 코드로 풀려면 보통 다음이 필요하다.

- 큐브 상태를 표현하는 방법
- `18개 action`을 적용했을 때 상태가 어떻게 바뀌는지
- 방문한 상태를 저장하는 자료구조
- 이전 상태와 사용한 action을 기록하는 자료구조

즉, 핵심은 "상태 표현", "18개 회전 구현", "최단경로 복원" 세 가지이다.
